### 기술 스택 상세

| 계층 | 기술 | 역할 |
|------|------|------|
| 프론트엔드 | Next.js 14 (App Router) | SSR + CSR 하이브리드 렌더링 |
| 언어 | TypeScript | 타입 안전성 |
| 스타일링 | Tailwind CSS + CSS Variables | 테마 시스템 (라이트/다크) |
| 폰트 | Pretendard Variable | 현대적 한글 폰트 |
| 애니메이션 | Canvas 2D + Framer Motion | 운명 애니메이션 + UI 전환 |
| 오디오 | Web Audio API | 프로시저럴 음악 생성 |
| AI | OpenAI GPT-4 | 사주 해석 + AI 상담 |
| DB | Supabase (PostgreSQL) | 사용자, 결제, 세션 |
| 인증 | Supabase Auth | 이메일/소셜 로그인 |
| 결제 | PortOne | 카드/간편결제 |
| 이메일 | Resend | 거래 이메일 |

## 4. 사주 계산 알고리즘

### 4.1 사주팔자 계산 흐름

```
입력: 양력 1996년 8월 4일 12시 (남성)
         │
         ▼
┌─────────────────────────┐
│ 1. 양력 → 음력 변환      │ ← 한국천문연구원 데이터 기반
│    1996.08.04 → 음력 변환 │
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│ 2. 년주 계산             │
│                         │
│    (연도 - 4) % 60       │
│    = 천간 인덱스 % 10    │
│    = 지지 인덱스 % 12    │
│                         │
│    1996: 병(丙) + 자(子)  │
│    → 년주 = 丙子 (화)     │
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│ 3. 월주 계산             │
│                         │
│    년간 × 2 + 월 기준    │
│    천간: (년간×2 + 월) % 10│
│    지지: 월에 대응 (고정)  │
│                         │
│    → 월주 = 乙未 (목)     │
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│ 4. 일주 계산             │
│                         │
│    기준일로부터의 일수 계산│
│    (2000-01-07 = 甲子)   │
│    diff = 날짜차이        │
│    idx = diff % 60       │
│    천간 = idx % 10       │
│    지지 = idx % 12       │
│                         │
│    → 일주 = 庚辰 (금)     │
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│ 5. 시주 계산             │
│                         │
│    일간 × 2 + 시간대     │
│    천간: (일간×2 + 시간) % 10│
│    지지: 시간대 대응 (고정) │
│                         │
│    → 시주 = 壬午 (수)     │
└────────────┬────────────┘
             ▼
결과: 丙子 乙未 庚辰 壬午
      화   목   금   수
```

### 4.3 오행 분포 계산

```
입력: 丙子 乙未 庚辰 壬午

각 글자의 오행:
  丙=화, 子=수, 乙=목, 未=토, 庚=금, 辰=토, 壬=수, 午=화

집계:
  목: 1 (乙)
  화: 2 (丙, 午)
  토: 2 (未, 辰)
  금: 1 (庚)
  수: 2 (子, 壬)

판정:
  강한 오행: 화 (또는 토, 수 — 동률)
  약한 오행: 목 (또는 금)
  균형 상태: "균형" (최대-최소 ≤ 1)
```

### 4.4 일간 강약 알고리즘

```
일간 = 庚 (금)

지지력 (Support):
  - 같은 오행(금): 庚 자체 = +1
  - 생해주는 오행(토→금): 未, 辰 = +2
  - 월지가 돕는가: 未=토 → 금을 생함 = +1
  합계: support = 4

극하는 힘 (Opposition):
  - 극하는 오행(화→금): 丙, 午 = -2
  - 설기하는 오행(금→수): 子, 壬 = -2
  합계: opposition = 4

점수: support - opposition = 0
판정: |점수| ≤ 2 → "중화" (또는 추가 규칙으로 "신약")
```

### 4.5 용신 판단 알고리즘

```
일간: 庚 (금)
강약: 신약 (점수 < 0)

규칙:
  신약한 일간을 돕는 오행이 용신
  금을 생하는 것 = 토 (토생금)
  금과 같은 것 = 금 (비겁)

용신: 토
희신: 금
기신: 화, 목 (금을 극하거나 설기)
```

## 6. 사주 음악 생성 알고리즘

### 6.1 오행 → 음악 매핑

```
┌──────────┬─────────┬──────────────────────────────┐
│ 오행     │ 음악 키  │ 특성                          │
├──────────┼─────────┼──────────────────────────────┤
│ 목 (木)  │ G major │ 성장하는 멜로디, 우드윈드 음색  │
│ 화 (火)  │ D major │ 강렬한 리듬, 빠른 템포         │
│ 토 (土)  │ C major │ 안정적 베이스, 중간 템포       │
│ 금 (金)  │ F major │ 맑은 종소리, 선명한 음색       │
│ 수 (水)  │ A minor │ 유려한 흐름, 느린 템포         │
└──────────┴─────────┴──────────────────────────────┘
```

### 6.2 4악장 구성

```
입력: 사주 결과 + 장르 선택

제1악장: 초년운 (년주 기반)
  - 기준 오행: 년주의 element
  - 템포: 기본 BPM × 장르 배수
  - 분위기: 시작, 씨앗

제2악장: 청년운 (월주 기반)
  - 기준 오행: 월주의 element
  - 템포: 약간 빠르게
  - 분위기: 성장, 도전

제3악장: 중년운 (일주 기반)
  - 기준 오행: 일주의 element
  - 템포: 절정
  - 분위기: 성취, 절정

제4악장: 말년운 (시주 기반)
  - 기준 오행: 시주의 element
  - 템포: 느려짐
  - 분위기: 결실, 안정
```

### 6.3 Web Audio 생성 파이프라인

```
┌─────────────┐
│ AudioContext │
└──────┬──────┘
       │
       ├── Pad (배경음)

       │       파형: 장르별 (sine/triangle/sawtooth)
       │       주파수: 오행 기준음
       │
       ├── Melody (멜로디)
       │   └── OscillatorNode → BandpassFilter → GainNode
       │       음계: 펜타토닉 [1, 9/8, 5/4, 3/2, 5/3]
       │       패턴: 랜덤 노트 시퀀스 (스케일 내)
       │       길이: 장르별 noteLength 범위
       │
       ├── Percussion (퍼커션)
       │   └── WhiteNoise → BandpassFilter → GainNode
       │       스타일: soft/crisp/heavy/shuffle/none
       │       스윙: 장르별 swingAmount
       │
       └── Effects
           ├── ConvolverNode (리버브)
           └── DelayNode (딜레이)

```
